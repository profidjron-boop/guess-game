## ADR-01: Режим гостя без логина (Supabase)

**Решение:** `player_id` = локальный UUID (`crypto.randomUUID`), профиль в `localStorage`, доступ к таблицам Supabase без регистрации.

**Плюсы:** быстрый вход, низкий порог для зрителей.  
**Минусы:** нет классической идентификации пользователя; RLS настроен под anon.

**Статус:** принято для v1.

---

## ADR-02: Очки и победитель раунда в БД

**Решение:** итог раунда и обновление `participants` выполняются в **`apply_round_results`** (и цепочке после **`mark_round_event`**).

**Плюсы:** одна истина на сервере; UI только отображает.  
**Минусы:** отладка цепочки RPC сложнее, чем чистый UI.

**Уточнение (2026-03):** `press_time_ms` считается в **`submit_guess_server`** на сервере; **`delta_ms`** после фиксации **`event_time_ms`** хостом. Клиент не является источником истины для времени и очков.

**Статус:** принято.

## ADR-03: Realtime по таблицам

**Решение:** подписки на изменения **`rooms`**, **`participants`**, **`rounds`** (фильтр по комнате).

**Статус:** принято.

## ADR-04: Сборка без env

**Решение:** обёртка Supabase-клиента не бросает исключение при отсутствии `NEXT_PUBLIC_*` во время `next build`.

**Статус:** принято.

## ADR-05: Эталон события на трансляции (хост)

**Решение:** в демо момент события на эфире фиксирует **хост** вызовом **`mark_round_event`**. До отметки `rounds.event_time_ms` и `guesses.delta_ms` могут быть `null`. После отметки — пересчёт дельт, **`apply_round_results`**, следующий раунд или **`finalize_game`**.

**Плюсы:** не нужен внешний таймкод для первой версии; единая серверная логика.  
**Минусы:** доверие к хосту как к «рефери» момента.

**Статус:** принято для демо.

## ADR-06: Без таймера «окна раунда»

**Решение:**

- В UI **нет** обратного отсчёта / полоски, ограничивающей приём нажатий по времени.
- В RPC **`submit_guess_server`** и **`mark_round_event`** **не** используется отсечение по **`rounds.duration_ms`** (поле остаётся в шаблонах/строках раундов как данные, не как жёсткий лимит механики).

**Плюсы:** меньше рассинхрона с реальным эфиром; игрок не упирается в «0.0 с» на полоске.  
**Минусы:** при необходимости анти-абуза в будущем понадобятся другие правила (например, только со стороны продукта).

**Статус:** принято.

---

## ADR-07: Человекочитаемые ошибки RPC

**Решение:** не полагаться на `String(unknown)` для ошибок Supabase; использовать **`formatUnknownError`** (`lib/formatError.ts`) перед выводом в UI.

**Статус:** принято.
