## Текущая механика (кратко)

1. Игрок смотрит **реальную трансляцию** на другом экране; в приложении — матч, команда, **`СЕЙЧАС!`**.
2. **Хост** фиксирует эталон момента события на эфире через RPC **`mark_round_event`** → задаётся `event_time_ms`, считаются дельты и очки.
3. До эталона у нажатия **`delta_ms = null`**; победитель раунда — по минимальному `|delta_ms|` среди строк с не-null дельтой.
4. **Отдельного таймера «окна раунда» в UI нет**; в RPC **нет отсечения** по `rounds.duration_ms` для нажатий и эталона (поле остаётся в данных из шаблонов).
5. Ошибки RPC в UI нормализуются через **`lib/formatError.ts`**, чтобы не показывать `[object Object]`.
